시스템 해킹 기초

1. 해킹의 기초

● 진(眞) 해킹：단순히 규칙을 창조적으로 활용해 의도치 않은 결과를 얻어내는 일련의 행위

해킹 ->
  • 웹 해킹 : 웹사이트 공격
  • 네트워크 해킹 : 네트워크 공격
  • 시스템 해킹 : 프로그램의 취약점 공격

● 소프트웨어 버그 : 프로그램이 잘못된 결과를 내거나, 오류를 발생하는 등의 의도치 않은 동작을 수행하게 되는 문제

● 소프트웨어 취약점 : 공격자가 주어진 권한 이상의 권한을 획득하거나 프로그래머가 의도하지 않은 동작을 수행할 수 있도록 하는 소프트웨어 버그

● 패치 : 수정 또는 개선을 위해 컴퓨터 프로그램이나 지원 데이터를 업데이트하도록 설계된 일종의 소프트웨어, 혹은 그러한 행위

□ 큰 프로그램이나 복잡한 시스템에서는 미처 발견하지 못한 취약점들이 존재할 수 있고, 공격자가 단 하나의 취약점만으로도 소프트웨어를 공격할 수도 있다.

취약점은 프로그래머의 실수, 잘못된 가정으로 발생한다.

2. 취약점이란?

● 익스플로잇 : 취약점을 이용해 공격자가 의도한 동작을 수행하게 하는 코드 혹은 이를 이용한 공격 행위


Bug : 프로그래머가 의도하지 않은 동작을 수행하는 것
Vulnerability : 보안에 영향을 미칠 수 있는 버그
Exploitable Vulnerability : 이를 이용해 공격자가 의도한 동작을 수행할 수 있는 버그
Reliably Exploitable Vulnerability : 매우 높은 확률로 공격에 성공할 수 있는 버그

● 모든 소프트웨어 취약점은 소프트웨어와 공격자가 상호 작용하는 곳(사용자의 입력)에서부터 발생


● Attack Vector : 공격자가 소프트웨어와 상호 작용할 수 있는 곳

● Attack Surface : Attack Vector들의 집합

● unreachable code 영역에 발생한 보안 취약점은 잠재적으로 악용 될 수 있다.

● 자연스럽게 취약점의 존재 여부와는 무관하게 프로그램을 보호하는 방법 --> 미티게이션(mitigation)
